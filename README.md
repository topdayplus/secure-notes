# 安全便签

安全便签是一款本地优先的加密便签应用，重点解决一个问题：敏感便签只加密保存在手机本地；换手机时，数据迁移可在断网状态下通过局域网传输完成，迁移包不落地保存，也可以导出为加密文件手动迁移。

## 核心能力

- 本地加密存储：便签内容加密后写入本地数据库。
- 启动密码与自动锁定：保护应用入口和后台切回场景。
- 断网局域网迁移：不依赖云服务，不需要互联网。
- 二维码配对：旧手机展示迁移二维码，新手机扫码获取迁移地址。
- 迁移确认：两台手机核对确认码和便签数量后再传输。
- 加密迁移包：局域网中传输的是加密数据，不是明文便签。
- `.snote` 文件兜底：局域网不可用时，可导出加密文件手动迁移。
- 导入进度显示：大批量便签导入时显示解密和写入进度。
- Android 隐私保护：防止截图和最近任务预览泄露。

## 离线迁移如何工作

1. 两台手机连接同一个局域网。
   - 可以是同一个 Wi-Fi。
   - 也可以是一台手机开热点，另一台手机连接该热点。
   - 不要求局域网能访问互联网。
2. 旧手机开启“局域网发送”，生成迁移地址和二维码。
3. 新手机扫码，获取旧手机的局域网迁移地址。
4. 两台手机核对确认码和便签数量。
5. 旧手机确认发送，新手机输入迁移口令后导入。

整个过程不会上传便签到云端。二维码只包含局域网地址，不包含便签内容。

## 适用场景

- 在便签中保存银行卡密码、账号密码、恢复短语或私人备忘。
- 不希望敏感内容经过云同步、网盘或第三方账号体系。
- 换手机时希望两台手机面对面完成数据迁移。
- 网络环境受限，但两台手机可以组成本地 Wi-Fi/热点局域网。

## 安全与隐私

- 应用不提供云同步。
- 便签内容加密后保存在本机。
- 迁移包使用迁移口令加密。
- 局域网迁移只传输加密迁移包。
- 相机权限只用于扫描局域网迁移二维码。

## 技术栈

- Flutter / Dart
- SQLite
- AES-GCM
- PBKDF2-HMAC-SHA256
- Android / iOS

## 当前状态

Android 端主要流程已可用，包括本地加密便签、启动密码、二维码扫码、断网局域网迁移和 `.snote` 文件迁移。

iOS 工程已存在，仍需要 macOS/Xcode 环境进一步验证。
