# 安全便签

定位是以安全性为主的便签，截屏防护、切屏保护，无网络局域网无落地文件同步数据等

## 为什么要做这项目

最近换不同牌子的安卓手机，之前的手机用的是系统内置的便签，而不同品牌内置的便签在软件搬家时并不会同步数据，真是离谱。之后也找到了一些开源的加密便签，可以提供导出加密数据再导入，但就是想能不能连导出都不需要，甚至不需要联网，**在断网状态下开启热点组件无网络的局域网，实现无文件落地传输**。

## 核心能力

- 断网局域网迁移：不依赖云服务，不需要互联网。
- `.snote` 文件兜底：局域网不可用时，可导出加密文件手动迁移。
- 隐私保护：防止截图和最近任务预览泄露。
- 本机数据清理：可在设置中清空当前手机上的所有便签。

## 离线迁移如何工作

1. 两台手机连接同一个局域网。
   - 可以是同一个 Wi-Fi。
   - 也可以是一台手机开热点，另一台手机连接该热点。
   - 不要求局域网能访问互联网。
2. 旧手机开启“局域网发送”，生成迁移地址和二维码。
3. 新手机扫码，获取旧手机的局域网迁移地址。
4. 两台手机核对确认码和便签数量。
5. 旧手机确认发送，新手机输入迁移口令后导入。

整个过程不会上传便签到云端。二维码只包含局域网地址，不包含便签内容。

## 适用场景

- 在便签中保存银行卡密码、账号密码、恢复短语或私人备忘。
- 不希望敏感内容经过云同步、网盘或第三方账号体系。
- 换手机时希望两台手机面对面完成数据迁移。
- 网络环境受限，但两台手机可以组成本地 Wi-Fi/热点局域网。

## 安全与隐私

- 应用不提供云同步。
- 便签内容加密后保存在本机。
- 迁移包使用迁移口令加密。
- 局域网迁移只传输加密迁移包。
- 相机权限只用于扫描局域网迁移二维码。

## 技术栈

- Flutter / Dart
- SQLite
- AES-GCM
- PBKDF2-HMAC-SHA256
- Android / iOS

## 当前状态

Android 端主要流程已可用，包括本地加密便签、启动密码、二维码扫码、断网局域网迁移和 `.snote` 文件迁移。

iOS 工程已存在，仍需要 macOS/Xcode 环境进一步验证。
